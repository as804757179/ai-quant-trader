# ADR-021：执行审批与订单意图边界

状态：Accepted

## 决策

交易请求不得自行声明已审批或已认证数据。服务端以订单规范化载荷 Hash、模式、请求主体、独立审批主体、有效期和已认证数据批次建立一次性执行授权。订单在调用券商前必须持久化为订单意图；审批只能由非请求者主体消费一次。

## 影响

- Worker、AI 和定时任务均不得绕过受认证 API 直接产单。
- 审批事件为追加式审计数据；审批、意图和 outbox 的当前状态只能按受控状态机推进，不删除历史记录。
- 审批、数据资格或意图不可验证时，交易必须拒绝；不以客户端字段或默认值放行。
- 实盘和发布安全锁不因本 ADR 或实现而改变，仍保持关闭。

## 验证与回滚

使用契约测试验证伪造审批字段、同人审批、过期/不匹配载荷、数据批次不合格和 Worker 旁路均被拒绝。迁移只追加，禁止 downgrade；若需要停用，仅关闭执行锁并保留审计数据。
