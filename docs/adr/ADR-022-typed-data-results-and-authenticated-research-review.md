# ADR-022：类型化数据结果与认证研究复核

状态：Accepted

## 决策

内部数据客户端必须以稳定状态、错误码、可重试标记和来源信息表达读取结果，不再把超时、上游失败、坏响应与无数据折叠为 `None`。展示层可保留兼容读取方法，但认证、研究、回测和执行路径不得把失败或无来源降级视为可用数据。

新闻标题/链接人工复核必须使用已认证 `research_reviewer` 主体；请求体不得声明复核人。每次追加复核必须携带 `Idempotency-Key`，并记录请求 Hash 与主体。

## 影响

- 未跟踪的新浪 K 线 fallback 已从后端 DataService 移除；上游故障明确返回空结果并记录原因。
- 既有新闻复核历史保留为历史审计记录；新记录绑定 Principal、幂等键和请求 Hash。
- 复核不改变 observed-only、Research Readiness、候选发布或订单权限。

## 验证与回滚

契约测试分别覆盖超时/坏响应/无数据、无新浪 fallback、最新条款优先、伪造复核人拒绝和幂等键冲突。迁移只追加认证字段，禁止 downgrade；如需停用入口，收回 `research:review.append` scope，不删除历史记录。
