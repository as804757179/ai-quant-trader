# ADR-013：多维研究证据观察语义

日期：2026-07-15  
状态：Accepted

## 背景

公告、新闻和财报在进入研究前需要保留来源、原文、发布时间、系统获取时间、内容 Hash 与可得时间。现有实时行情血缘仅覆盖 `market.quotes`，不能证明多维研究信息在某个历史时点已经公开且已被系统获得。

## 决策

1. 新增独立的研究证据观察 sidecar，不写入 Certified Store，不修改现有 Research Readiness 审核，也不产生候选、回测输入或订单。
2. 每条证据记录 Provider、source、发布者、原文 URL、外部文档 ID、原文 Hash、采集与标准化版本、批次与失败原因。相同外部文档 ID 出现新 Hash 时追加记录，不覆盖旧证据。
3. 分开记录 `source_published_at`、`source_published_date`、时间精度、`first_observed_at` 与 `available_at`。当来源只提供日期而非精确公开时间时，`source_published_at` 保持为空，`available_at` 仅取系统首次成功观测时间。
4. Provider 或时间语义不完整、原文下载或 Hash 失败、来源与请求标的不一致时写入 rejected 审计，不能用 fallback、Mock、Synthetic、上一版本内容或推测时间补齐。
5. Sprint14.3 试点固定为巨潮资讯的上市公司公告。巨潮的公开页面为法定信息披露平台，但其自动化使用许可仍标记为 `review_required`；本阶段只做显式人工触发的受控采集，不增加定时抓取。
6. 2026 年沪深交易日历按两所公布的全年休市安排补齐。该覆盖只消除当前日期的日历缺口；不会自动提升任何行情或研究证据的 Readiness。

## 后果

- 只读 API 可以解释一条公告来自何处、何时被系统观察、原文是否可校验，以及为什么它不可交易。
- 日期精度公告不会被用于任何盘中或历史时点推断；后续若取得精确时间戳，必须新增带明确来源与版本的证据。
- 新闻与财报可复用同一模型，但在各自固定 Provider、许可、时间语义和失败策略通过设计评审前不得接入。
- 现有六个发布/交易锁和 Execution Gate 不变。

## 回滚

回滚本 ADR 对应代码会停止显式公告采集与只读展示。已写入的研究证据和失败批次是审计事实，不应通过删除记录伪造未发生；数据库回滚需单独评审数据保留方案。交易日历仅删除本 ADR 新增且来源引用匹配的行，不改写既有日历记录。
