## 1. 产品目标与边界

目标是逐步建设面向 A 股的自动化**模拟交易**系统：

```text
真实且可追踪的数据
→ 数据认证
→ Research Readiness
→ 可信回测
→ 策略验证
→ 影子运行
→ 自动模拟交易
→ 未来受控小资金实盘
```

当前不接入真实资金；模拟资金、市场规则、交易成本、T+1、涨跌停、整手/零股、停牌和企业行动必须按真实 A 股规则处理。

最终业务闭环：持续监控全市场行情、公告、新闻、财报、行业、情绪、订单回报与风险事件；自动筛选机会并复核当前持仓；在不操作、建仓、加仓、减仓、清仓换股之间做组合决策；交易后记录订单、成交、持仓、现金、费用、盈亏与复盘；复盘只提出优化候选，不得自动上线策略。

不承诺盈利。可运行代码、正确回测或小样本盈利，均不等于策略具备持续盈利能力。

## 2. 不可绕过的安全与数据边界

- AI 只能分析、解释、评分或输出 recommendation，不能直接或间接创建订单。
- Celery 定时任务默认不能产生订单。
- 所有订单必须经过 Data Certification、Research Readiness、Risk Engine、Execution Gate 和范围化授权/审批。
- `unknown`、`synthetic`、`uncertified` 数据不得进入可信研究或交易链路。
- `market.klines` 是 legacy/raw/uncertified 审计数据；`market.certified_klines` 是认证历史数据存储。
- 历史研究数据统一通过 `CertifiedKlineRepository` 读取。
- Certified 不等于所有用途均 ready；权限按股票、日期范围、用途与 Requirement Profile 授予，不能传播。
- 禁止未来函数；第 t 日收盘后生成的信号最早下一交易日执行。
- Live 初始化失败不得静默降级为 Mock 或 Paper。

当前六个发布与交易锁必须保持关闭：

```text
CERTIFIED_BACKTEST_EXECUTION_ENABLED=false
CERTIFIED_SCREENER_OUTPUT_ENABLED=false
TRADING_EXECUTION_ENABLED=false
LIVE_TRADING_ENABLED=false
AI_ORDER_ENABLED=false
ALLOW_SCHEDULED_ORDER=false
```

这些是当前安全默认值，不是永久禁令。未来如用户明确授权开放相应能力，必须在同一任务中说明原因、影响范围、ADR、测试、验收和回滚方案。
