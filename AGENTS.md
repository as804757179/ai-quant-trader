# 项目代理规则

本项目的优先级是：正确、安全、简单、可验证。任何修改都必须遵守以下规则。

## 工作方式

1. 编码前先说明目标、成功标准、最小方案与取舍。需求存在会改变结果的歧义时，先询问，不猜测。
2. 只修改完成当前目标所需的行，保持现有风格；不顺手重构，不处理无关问题，不新增未经确认的第三方依赖。
3. 所有说明、过程更新和脚本日志使用中文；代码注释使用英文。
4. 修改代码时只展示差异或具体片段，不重印整个文件。
5. 开始工作时先运行 scripts\doctor.ps1，并检查 git status；保留用户已有的未提交修改。

## 防卡死约束

1. 禁止直接运行 npm run dev、Vite、Uvicorn、Celery worker/beat、docker compose up 或 docker compose logs --follow。
2. 启动项目只能运行 scripts\start-dev.ps1 或 scripts\start-local.ps1；停止只能运行 scripts\stop-local.ps1。
3. 依赖准备只能显式运行 scripts\bootstrap.ps1，启动脚本不得默认安装依赖。
4. 所有可能阻塞的命令必须有明确超时；长任务至少每 60 秒报告一次状态。
5. 文件搜索优先使用 rg，并遵守忽略规则；禁止对虚拟环境、node_modules、运行日志或仓库根目录做无界递归扫描。
6. 不得按端口或模糊进程名批量终止进程。只允许停止运行登记中且 PID、启动时间、命令指纹均匹配的项目进程。
7. 除非用户明确要求，不启动子代理或并行代理。

## 量化安全边界

不得在未明确授权时改变行情数据语义、回测口径、策略逻辑、风险限制、组合约束、订单边界、实盘开关或安全默认值。验证结果必须来自真实命令，不得用合成通过结果代替。
